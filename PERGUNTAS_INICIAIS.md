# Questões iniciais acerca da compreensão e funcionamento do programa

## 1. O que significa ganhar o jogo?

Ganhar um jogo é quando uma pessoa consegue ganhar alguma coisa ou na totalidade do jogo que pode ser uma partida online, conclusão de um mapa de jogo, etc.

## 2. O que significa perder o jogo?

Perder um jogo é quando uma pessoa não consegue alcançar um objetivo num jogo, seja porque perdeu uma partida online ou porque perdeu alguma coisa num mapa, etc.

## 3. O que torna uma tentativa válida?

Uma tentativa válida é quando o utilizador introduz algum valor, seja texto ou números e que cumpre com os requesitos de uma determinada variável. Podem ser requesitos de sistema ou elaborados pelo utilizador.

## 4. O que torna uma tentativa inválida?

Uma tentativa inválida é quando o utilizador introduz algum valor que não cumpre com os requesitos necessários de uma determinada variável. Podem ser requesitos de sistema ou elaborados pelo utilizador.

## 5. Uma tentativa inválida deve gastar uma tentativa do jogador? Porquê?

Depende. Se for um jogo, onde um utilizador perde vidas quando introduz algum valor inválido, deve subtrair (tirar) uma vida ao utilizador porque inseriu um valor que não é o que se pretende. Caso for um jogo que não perde vidas, não faz sentido tirar vidas onde não existe (por outras palavaras, não é para tirar vidas a uma variável que não existe).

## 6. Que informação o jogador precisa de ver depois de cada tentativa?

Para este jogo, o utilizador precisa de saber o número de tentativas restantes, dizer se a posição do número que foi introduzido pelo utilizador está correta ou não e saber quantas letras estão certas e quantas estão erradas. Quando as tentativas esgotarem, o utilizador precisa de saber qual era o número.

## 7. Que informação o programa precisa de guardar, mesmo que não a mostre sempre?

O programa precisa de guardar o código gerado de forma aleatória.
